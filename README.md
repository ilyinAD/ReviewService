# Как запускать?

Сначала генерация по спецификации: oapi-codegen -config api/config.yaml api/openapi.yaml

Запустить команду: docker-compose up из docker-deploy

# Архитектура

По спецификации генерируем описанные ручки (spеcification first подход) и структуры запросов и ответов \
Далее идет слоеная архитектура - сначала запрос принимает нужный handler и передает его слою usecase-ов \
usecase использует один или несколько репозиториев для получения данных из базы данных и возвращает результат или ошибку. \
handler оборачивает ошибку в тот вид который нужен для API, либо возвращает ответ, при отсутствии ошибки. \
В usecase также используются транзакции для правильного выполнения сложных операций таких как создание ПР например\
Миграции в dockеr-composе делаются через liquibase-migrations migrations
